# 中级全栈工程师学习计划

目标：从"能独立完成功能"到"能设计系统、能协作开发、能扛住生产环境"。

预计时间：8-12 周。

---

## 第 1-3 周：React 前端框架

目标：用 React 重写记账应用前端。

学习内容：

- JSX 语法、组件、props、state
- useState、useEffect
- React Router（页面路由）
- Axios 替代 fetch
- Tailwind CSS 或 Ant Design（UI 库）

项目：

- 用 React 重写 web_ledger.html
- 拆分组件：LoginForm、RecordTable、StatsPanel、AddRecordForm
- 状态管理：token 存 context，数据用 useState

检查点：功能与旧版一致、组件拆分合理。

---

## 第 4-5 周：Git 协作 + 工程化

目标：能参与多人项目。

学习内容：

- 分支策略：feature branch → PR → merge
- Code Review：提交 PR、收到 review 意见、修改
- rebase vs merge
- Git 回滚：reset、revert
- npm 项目：package.json、node_modules、.gitignore

练习：

- 给自己开 3 个分支，分别加功能和修 bug，各自提 PR 合并
- 模拟一次 merge conflict 并解决

---

## 第 6-7 周：PostgreSQL + 数据库进阶

目标：从 SQLite 升级到工业级数据库。

学习内容：

- 安装 PostgreSQL、pgAdmin
- 数据类型、索引、外键约束
- 连接池（psycopg2-pool 或 SQLAlchemy ORM）
- 数据库迁移（Alembic）
- 慢查询分析（EXPLAIN）

项目：

- 将记账应用的 SQLite 换成 PostgreSQL
- 加外键约束：records.user_id → users.id
- 给 date 和 user_id 建联合索引
- 用 Alembic 管理表结构变更

---

## 第 8-9 周：Docker + 环境管理

目标：代码在任何机器上跑起来都一样。

学习内容：

- Dockerfile、docker-compose
- 多容器编排（app + db）
- 环境变量注入
- 数据卷挂载

项目：

- 写 Dockerfile 打包 Flask 后端
- docker-compose.yml 编排 Flask + PostgreSQL
- 用 .env 文件管理密钥
- 一键启动：docker-compose up

---

## 第 10 周：日志 + 监控 + 错误处理

目标：线上出 bug 能快速定位。

学习内容：

- Python logging 模块（INFO / WARNING / ERROR 分级）
- 结构化日志（JSON 格式）
- Sentry 错误追踪（免费版）
- 全局异常捕获

项目：

- 给后端所有接口加 request_id + 请求日志
- 接入 Sentry，故意抛错验证能不能收到

---

## 第 11 周：性能优化基础

目标：知道慢在哪、怎么修。

学习内容：

- 接口性能分析（timeit / cProfile）
- N+1 查询问题
- 缓存基础（Redis）
- 前端：懒加载、防抖、虚拟列表

项目：

- 给统计接口加模拟数据（1000 条），分析耗时
- 加索引后对比性能
- 给登录 token 校验加 Redis 缓存（可选）

---

## 第 12 周：项目整合作品

目标：一个能放进简历的中级项目。

功能：

- React 前端 + FastAPI 后端
- PostgreSQL 数据库
- Docker 部署
- GitHub Actions CI（自动跑 pytest）
- Sentry 错误监控
- README + 架构图

---

## 每周固定节奏

- 代码：每周完成目标项目功能
- AI：用 AI 帮你解释新概念、review 代码、优化实现
- 记录：每周复盘写了什么、卡在哪里
- 仓库：所有代码推到 GitHub

---

## 核心判断标准

中级 vs 初级的本质区别不是你写了多少功能，而是：

| 初级想的是 | 中级想的是 |
|-----------|-----------|
| 这段代码怎么跑通 | 这段代码以后别人怎么改 |
| 一个接口写完了 | 接口挂了怎么排查 |
| 数据库能存就行 | 表结构经不经得住 10 万条数据 |
| 自己能用就行 | 换个人来能不能跑起来 |

每学完一个阶段，问自己：如果明天交给别人维护，他能看懂吗？
