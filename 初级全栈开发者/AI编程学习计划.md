# AI 辅助编程 12 周学习计划

目标：用 AI 辅助学习编程，最终完成一个真实可运行的 Web 项目，并能理解关键代码、调试问题、整理作品。

默认路线：Python + Web + 数据库 + AI 辅助开发。

---

## 每天固定节奏

建议每天学习 1.5 到 2 小时：

1. 20 分钟：学习一个小知识点
2. 50 分钟：自己敲代码练习
3. 30 分钟：用 AI 帮你解释、改错、扩展
4. 10 分钟：写学习记录

每天记录 3 件事：

- 今天学了什么
- 今天卡在哪里
- 明天继续做什么

重要规则：

- AI 可以帮你写代码，但你必须读懂关键代码。
- 不要只复制粘贴，要自己运行、修改、调试。
- 每段关键代码都问 AI：这段代码为什么这样写？有没有更简单的写法？

---

## 第 1-2 周：Python 编程基础

目标：能写简单 Python 程序。

学习内容：

- 变量、字符串、数字
- if 判断
- for / while 循环
- list、dict
- 函数
- 文件读写

练习项目：

- BMI 计算器
- 猜数字游戏
- 学生成绩统计程序
- 文本记账本，把收入和支出保存到文件

推荐 AI 提问：

```text
我是初学者，请用 Python 教我 list 和 dict。先解释概念，再给 5 个练习题，不要直接给答案。
```

---

## 第 3-4 周：小项目和调试

目标：能独立写出命令行小工具。

学习内容：

- 函数拆分
- try / except 错误处理
- import 模块
- 看懂报错
- Git 基础

练习项目：

- 命令行 Todo 工具
- 命令行记账工具
- 文件批量重命名工具
- CSV 成绩分析工具

你要练会：

- 看报错
- 复制报错问 AI
- 让 AI 解释错误原因
- 自己修改后重新运行

推荐 AI 提问：

```text
下面是我的代码和报错。请不要直接重写全部代码，只告诉我错误原因、最小修改方案，以及如何避免类似错误。
```

---

## 第 5-6 周：Web 基础

目标：知道网页和后端怎么交互。

学习内容：

- HTML 基础
- CSS 基础
- JavaScript 基础
- HTTP 请求
- Flask 或 FastAPI 入门

练习项目：

- 个人主页
- 网页计算器
- 网页 Todo
- 用 FastAPI 写 3 个接口：新增、查询、删除

推荐 AI 提问：

```text
请用生活化方式解释 HTTP、前端、后端、API、数据库之间的关系，然后给我一个最小可运行示例。
```

---

## 第 7-8 周：数据库和完整后端

目标：能做一个带数据保存的小系统。

学习内容：

- SQL 基础
- SQLite
- 表设计
- CRUD：增删改查
- 用户注册和登录基础
- 密码哈希概念

练习项目：

- 记账系统后端
- Todo 系统后端
- 书籍管理系统
- 简单用户注册登录

推荐 AI 提问：

```text
我想设计一个记账系统数据库，有用户、账目、分类。请先帮我设计表结构，并解释每个字段为什么需要。
```

---

## 第 9-10 周：完整项目实战

目标：做出一个能展示的作品。

推荐项目：个人记账 Web 应用。

功能要求：

- 用户注册 / 登录
- 新增收入 / 支出
- 查看账目列表
- 按分类筛选
- 按月份统计
- 简单图表
- 数据保存到 SQLite
- README 说明如何运行

开发顺序：

1. 画功能列表
2. 设计数据库
3. 写后端接口
4. 写前端页面
5. 前后端联调
6. 加错误提示
7. 加测试
8. 写 README

推荐 AI 提问：

```text
我正在做个人记账 Web 应用。请你作为技术导师，每次只带我完成一个小步骤。完成后给我检查点，不要一次性生成整个项目。
```

---

## 第 11 周：测试、重构、工程化

目标：让项目不只是能跑，而是更可靠。

学习内容：

- 单元测试
- 接口测试
- 代码重构
- 环境变量
- requirements.txt
- README
- GitHub 基础

练习任务：

- 给注册登录写测试
- 给新增账目写测试
- 把重复代码提取成函数
- 写清楚项目启动步骤

推荐 AI 提问：

```text
请帮我 review 这段代码，重点看 bug、安全问题、重复代码和可测试性。先指出问题，不要急着改代码。
```

---

## 第 12 周：部署和作品集

目标：把项目变成可以展示的作品。

学习内容：

- GitHub
- 部署基础
- Render / Railway / Vercel / PythonAnywhere 任选一个
- 项目截图
- 项目介绍

最终成果：

- 一个 GitHub 仓库
- 一个 README
- 一个可运行项目
- 一段项目介绍
- 你能讲清楚项目怎么设计、怎么运行、哪里最难

推荐 AI 提问：

```text
请帮我把这个项目整理成作品集介绍，包括项目亮点、技术栈、功能列表、我学到了什么，以及面试时怎么介绍。
```

## 每周复盘问题

每周最后一天回答这 5 个问题：

1. 这周我完成了什么？
2. 我真正理解了什么？
3. 哪些代码是 AI 写的但我还没看懂？
4. 我遇到的最大 bug 是什么？
5. 下周我要补什么？

---

## 使用 AI 的正确方式

好的用法：

- 让 AI 解释概念
- 让 AI 帮你拆任务
- 让 AI 找 bug
- 让 AI review 代码
- 让 AI 给练习题
- 让 AI 帮你比较方案

不好的用法：

- 完全复制 AI 代码但不运行
- 报错后直接让 AI 重写整个项目
- 一次性让 AI 生成完整大项目
- 不看文档、不看代码、不做复盘

核心判断标准：

如果 AI 关掉之后，你还能解释项目怎么运行、数据怎么流动、哪里可能出错，说明你真的学到了。
